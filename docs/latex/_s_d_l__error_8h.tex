Retrieve a message about the last error that occurred on the current thread.

It is possible for multiple errors to occur before calling \mbox{\hyperlink{_s_d_l__error_8h_adaef317de186f072f8328798de29d51a}{SDL\+\_\+\+Get\+Error()}}. Only the last error is returned.

The message is only applicable when an SDL function has signaled an error. You must check the return values of SDL function calls to determine when to appropriately call \mbox{\hyperlink{_s_d_l__error_8h_adaef317de186f072f8328798de29d51a}{SDL\+\_\+\+Get\+Error()}}. You should {\itshape not} use the results of \mbox{\hyperlink{_s_d_l__error_8h_adaef317de186f072f8328798de29d51a}{SDL\+\_\+\+Get\+Error()}} to decide if an error has occurred! Sometimes SDL will set an error string even when reporting success.

SDL will {\itshape not} clear the error string for successful API calls. You {\itshape must} check return values for failure cases before you can assume the error string applies.

Error strings are set per-\/thread, so an error set in a different thread will not interfere with the current thread\textquotesingle{}s operation.

The returned string is internally allocated and must not be freed by the application.

\begin{DoxyReturn}{Returns}
a message with information about the specific error that occurred, or an empty string if there hasn\textquotesingle{}t been an error message set since the last call to \mbox{\hyperlink{_s_d_l__error_8h_a007ed517953a99a46e46be567ef5609e}{SDL\+\_\+\+Clear\+Error()}}. The message is only applicable when an SDL function has signaled an error. You must check the return values of SDL function calls to determine when to appropriately call \mbox{\hyperlink{_s_d_l__error_8h_adaef317de186f072f8328798de29d51a}{SDL\+\_\+\+Get\+Error()}}.
\end{DoxyReturn}

This function is available since SDL 2.\+0.\+0.
